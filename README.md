
# BookVerse

> The BookVerse website is a web platform for an online bookstore that allows users to view, search, and purchase various books and publications, including scientific books, textbooks, magazines, fiction, and promotional packages. The primary goal of the site is to provide a simple, well-organized, and intuitive experience for users who wish to purchase books and publications online.
> 
> 



## 1. Core Features

The platform provides the following functionalities:

* Displaying products organized by categories.


* Advanced search and filtering of products.


* User authentication.


* A dynamic discount system.


* A system for creating custom publication packages.


* Inventory management.


* A well-structured review system.


* Personalized suggestions based on the customer's previous purchases.


* A Trade-In system designed for second-hand publications.

## 3. Site Architecture

The platform's structure is divided into specific pages:

### 3.1. Home Page

This serves as the presentation page of the online bookstore and the main navigation hub. It includes:

* General presentation of the store, popular products, and an offers section.


* Main categories (Publications, Packages, Trade-In) and a search bar.


* "About Us" section, contact details, and customer support in the footer.


* Quick access links to the Products, Cart, Account, and Trade-In pages.



### 3.2. Products Page

Dedicated to listing the available publications. Features include:

* Listings of books and magazines with filtering options by category, price, and author, alongside a search function.


* "Add to cart" and "View details" buttons directly on the listing.


* Product details are displayed in a modal window (popup) to avoid user redirection and improve navigation. This window contains the full description, reviews and ratings, additional information, and an add to cart button.



### 3.3. Cart & Checkout Page

* 
**Cart Section:** Displays the list of added products, modifiable quantities, individual prices, applied discounts, general total, and buttons to "Continue shopping" or "Checkout".


* 
**Checkout Section:** Displayed dynamically on the same page, including delivery details, delivery method, simulated payment method, order summary, and the "Place order" button. This keeps the buying process efficient without requiring an extra page.



### 3.4. User Account Page

Authentication and registration are handled via a modal window accessible from the site's header. The user dashboard contains:

* Order history.


* Personal data management.


* Personalized suggestions based on previous orders.


* Logout option.



### 3.5. Trade-In Page

A dedicated page for the second-hand publication exchange service. It includes:

* An explanation of how the system works and the steps of the evaluation process.


* A form for submitting data about the publication and request confirmation.


* This service has a separate page due to its distinct functionality.
